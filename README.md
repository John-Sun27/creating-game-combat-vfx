# 游戏战斗特效（2D）制作 Skill

[简体中文](README.md) | [English](README.en.md)

这是一个可复用的 chatgpt Skill，用于将游戏战斗特效从视觉定位推进到可接入、可配置、可测试和可验收的最终状态。流程已经在剑、火、水三类技能中完成验证，覆盖视觉设计、序列帧资源制作、语义化播放、配置导出、隐藏 GM 测试、回归测试和真机视觉验收。

## 项目目标

游戏特效需要根据游戏风格、技能的实际效果--攻击方向、范围等来提出需求进行设计制作。本 Skill 将战斗机制和视觉表现拆分为独立系统，把技能设计转成为可视化效果，再根据效果一步步制作成可用特效，最终接入项目中以实现特效的高效制作。


## Skill 覆盖内容

- 基于项目自身风格的视觉定位与四阶段分镜
- 透明 PNG 序列帧、帧数、锚点、透明度和曝光规范
- 近身剑气、飞行物、多弹体、移动浪头、天降物、地面喷发、持续领域、陷阱、烙印、光束和护盾环绕等播放模型
- 移动主体、固定地面锚点、局部尾迹与带时间戳命中事件的状态拆分
- 由配置表控制大小、偏移、宽度、半径以及导表结果验证
- 用隐藏手势呼出 GM 面板，独立播放任意技能
- 特效渲染测试与完整战斗机制回归测试
- 模拟器或真机关键帧截图及最终验收标准

## 七阶段工作流

1. **需求分析：** 记录技能语义、目标范围、项目约束，以及现有资源、渲染器、配置、测试和调试入口。
2. **视觉设计：** 确定美术方向、四阶段分镜和验收标准；默认在本阶段完成后暂停，等待确认。
3. **资源制作：** 根据已确认的设计制作透明序列帧、分层资源和特效清单。
4. **资源预览：** 在本地预览器中检查资源、记录问题和调整决定；默认在本阶段完成后再次暂停，等待确认。
5. **游戏接入：** 将已通过预览的资源接入运动模型、锚点、生命周期事件和权威调节项，不修改战斗机制。
6. **测试工具：** 增加独立的 GM/调试播放入口，并验证配置导出。
7. **验收优化：** 完成视觉验收和战斗回归测试，修正问题并交付最终报告。

## 使用方法

将本目录安装或链接到 Codex 的 Skills 目录，然后调用：

```text
使用 $creating-game-combat-vfx 为这组技能设计、制作并接入完整战斗特效。
```

普通使用者无需复制模板、填写 JSON 或执行命令。只需描述目标、范围和是否需要接入游戏；Skill 会判断已有成果与依赖，提出相关阶段并等待你选择，而不是不加区分地执行整套流程。

## 使用示例

### 完整请求

用户只需提供目标与范围：

> 使用 `$creating-game-combat-vfx` 为剑、火、水三组技能设计完整战斗特效。先完成视觉设计，再制作非程序化序列帧资源并接入游戏；保留现有伤害机制，增加可调大小与偏移，并通过隐藏 GM 面板逐个验收。

收到完整请求后，Skill 的第一次回复会列出可选择的七个阶段，并标记每个阶段是“已满足”“建议执行”还是“缺少依赖”：

1. 需求分析
2. 视觉设计
3. 资源制作
4. 资源预览
5. 游戏接入
6. 测试工具
7. 验收优化

你可以直接回复：

- `执行全部`
- `先完成前 3 步`
- `只做第 2、4 步`
- `从第 5 步继续`

选择 `执行全部` 时，流程仍会在第 2 步视觉设计和第 4 步资源预览后暂停确认；只有明确要求不中断执行时，才跳过这两个检查点。继续后续阶段时，Skill 会复用已检查的设计和资源，只补齐必要依赖。

### 局部请求

如果只需要某一部分，可以直接提出，例如：

> 使用 `$creating-game-combat-vfx` 只预览现有特效资源。

Skill 会直接进入第 4 步资源预览；如果缺少可读取的资源或清单，只会说明并补充最少的必要依赖，不会重复已经完成的视觉设计或资源制作。

### 启动本地预览

双击 `tools/vfx-preview/index.html`用浏览器打开，然后在页面中选择包含特效清单和 PNG 资源的目录。项目的构建或导出流程应自动在运行时资源旁生成 `effect-manifest.preview.json`，无需手动复制或维护第二份配置。无需安装依赖、启动服务器或联网；预览器会隔离显示缺失资源等问题，不影响同一目录中其他有效特效的播放。

预览器优先采用清单中的项目默认展示方式，包括运动类型、方向、距离、持续时间、锚点和分层时间轴。需要检查其他项目或临时比较时，可以用“展示方式、方向、距离”等控件进行手动覆盖；手动覆盖只影响当前预览，不会修改清单或 PNG 文件，切换特效或点击“恢复项目默认”即可清除。预览结果用于资源阶段检查，最终效果仍以游戏内实际比例、相机和混合方式下的验收为准。

每个制作阶段开始前，根据 `SKILL.md` 的指引读取对应参考文件。只有自动化战斗机制测试和最终游戏比例截图验收都通过后，才能将特效标记为完成。

## 设计原则

保持战斗机制权威，视觉表现可替换：

- 战斗 Actor 负责目标、碰撞、伤害、穿透、持续次数、Buff 和治疗；
- 视觉 Actor 负责位置、地面锚点、尾迹、天降高度、大小和偏移；
- 视觉事件负责带时间戳的命中、脉冲、破碎和一次性播放生命周期。

这套拆分方式可以在保证战斗机制稳定的同时，让特效真正按照视觉设计播放。
