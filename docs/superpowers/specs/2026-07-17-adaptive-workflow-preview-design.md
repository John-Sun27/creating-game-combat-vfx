# 自适应战斗特效流程与通用预览器设计

## 目标

将 `creating-game-combat-vfx` 从固定的全流程执行器升级为可选择、可暂停、可回退的阶段化助手。Skill 根据用户需求和项目现状推荐必要阶段，允许执行全部、前 N 步、指定步骤或从某一步继续，并在资源接入游戏前提供本地通用预览器。

## 默认交互

### 完整需求

当用户提出“为我的技能设计完整战斗特效”等完整需求时，Skill 先返回适配当前项目的阶段列表，不直接开始制作。回复必须包含：

- 推荐阶段及每个阶段的一句话产物说明；
- 已满足、需要执行和因依赖暂不可执行的阶段；
- 可直接回复的选择示例：`执行全部`、`先完成前 3 步`、`只做第 2、4 步`、`从第 5 步继续`；
- 推荐选择及原因。

### 明确的局部需求

当用户明确要求“只做视觉设计”“预览现有资源”“接入已有特效”或“只检查曝光问题”时，Skill 直接进入对应阶段。只有缺少必要输入或前置产物时，才补充最少的依赖阶段并说明原因。

### 用户选择语义

- `执行全部`：将七个阶段纳入当前目标，但仍在视觉设计和资源预览两个验收点暂停。
- `先完成前 N 步`：完成第 1 到第 N 阶段后停止，并汇报继续方式。
- `只做第 X、Y 步`：只执行指定阶段；缺少强依赖时先请求补做必要阶段。
- `从第 N 步继续`：复用已存在并通过检查的前置产物，从指定阶段开始。
- 只有用户明确要求“连续执行，不需确认”时，才跳过两个默认验收暂停点。

## 七阶段模型

| 阶段 | 主要输入 | 主要产物 | 默认暂停 |
|---|---|---|---|
| 1. 需求分析 | 用户目标、技能机制、项目状态 | 技能语义、目标范围、资源与接入现状 | 否 |
| 2. 视觉设计 | 阶段 1 结果、项目美术风格 | 风格定位、四阶段分镜、验收标准 | 是 |
| 3. 资源制作 | 批准的视觉设计 | 非程序化 PNG 序列帧、分层资源、内部特效清单 | 否 |
| 4. 资源预览 | 资源目录、内部特效清单 | 本地预览结果、资源问题列表、调整结论 | 是 |
| 5. 游戏接入 | 通过预览的资源、运行时结构 | 运动模型、锚点、生命周期与配置接入 | 否 |
| 6. 测试工具 | 已接入技能、现有调试入口 | GM 调用、单技能回放、导表验证 | 否 |
| 7. 验收优化 | 完整运行结果 | 视觉验收、机制回归、问题修正与最终报告 | 否 |

阶段 3 依赖阶段 2 的批准结果；阶段 4 依赖可读取的资源；阶段 5 依赖通过预览或被用户明确接受的资源。若用户已有合格资源或设计稿，可由 Skill 检查后视为对应前置阶段已完成。

## 阶段完成反馈

每个阶段结束时，Skill 输出：

1. 本阶段完成的产物；
2. 发现的问题与已做调整；
3. 当前可执行的下一阶段；
4. 可直接回复的操作，例如 `调整视觉方向`、`重新制作飞行主体`、`批准并继续第 5 步`、`暂时停止`。

反馈以决策信息为主，不要求用户理解内部清单、脚本或命令。

## 通用本地预览器

### 位置与启动

预览器位于 `tools/vfx-preview/`。用户直接打开 `index.html`，通过目录选择器载入一个包含内部特效清单和 PNG 资源的输出目录。页面不依赖构建工具、包管理器、本地服务器或网络资源。

### 数据来源

预览器复用 `assets/effect-manifest.example.json` 所定义的清单结构。资源路径相对于用户选择的目录解析。运动逻辑由 `visualArchetype` 驱动；帧数、大小和偏移沿用清单字段。预览器不维护第二份业务配置。

### 界面组成

- 左侧：特效列表、元素和运动类型；
- 中间：预览舞台与完整生命周期播放；
- 右侧：播放、暂停、重播、逐帧、帧率、大小、X/Y 偏移、循环方式和背景控制；
- 底部：当前阶段、当前帧、资源检查结果和错误提示。

### 播放行为

预览器按 `预警 → 主体运动 → 命中 → 残留` 播放完整生命周期，并根据运动类型模拟：

- 近身剑气；
- 单体飞行物与多弹体；
- 移动浪头或道路攻击；
- 天降物；
- 地面喷发；
- 持续领域；
- 陷阱、目标烙印、光束和护盾环绕。

飞行主体保持固定尺寸并移动，尾迹跟随主体；天降主体从舞台上方进入且地面预警固定；持续领域循环到退出阶段。页面使用 PNG、DOM 和 CSS 变换进行显示，不使用程序化 Canvas 绘制特效。

### 资源检查

载入时检查并在页面中列出：

- 清单缺失、JSON 无效或字段不完整；
- PNG 文件缺失；
- 帧数小于对应运动类型的最低要求；
- 横向序列帧宽度不能被帧数整除；
- PNG 不包含透明通道；
- 未知运动类型或无法生成生命周期。

单个特效出错时只阻止该特效播放，其他特效仍可预览。错误信息包含资源名、原因和建议修正方式。

### 预览验收

预览完成后，Skill 汇总检查结果并提供：

- `调整资源`：回到阶段 3，保留问题列表；
- `批准并进入游戏接入`：进入阶段 5；
- `暂时停止`：保留当前阶段状态，等待后续继续。

## Skill 与文档调整

- `SKILL.md`：增加需求路由、七阶段选择规则、依赖检查、默认暂停点和阶段完成反馈。
- `references/preview-workflow.md`：记录预览器载入、运动映射、错误处理与验收规则。
- `tools/vfx-preview/`：提供无构建依赖的本地预览页面及可测试的纯逻辑模块。
- `README.md` 与 `README.en.md`：增加分步调用、局部功能调用和“先完成前 3 步”的示例。
- `agents/openai.yaml`：确认默认调用语句仍能触发完整需求路由，必要时同步更新。

## 测试与验收

- 文档行为测试验证完整需求会先展示阶段选择，而局部需求会直接进入对应阶段。
- 纯逻辑测试覆盖清单解析、资源路径、帧宽校验、最低帧数和全部运动类型的生命周期生成。
- 浏览器验收覆盖目录载入、特效切换、播放控制、参数调整、错误隔离和三类代表性运动：飞行、天降、持续领域。
- Skill 结构校验和现有特效清单校验继续通过。
- 中英文说明对阶段编号、暂停点、示例回复和预览器能力保持一致。

## 不在本次范围

- 预览器不写回游戏配置表；调整值由 Skill 在用户批准后同步到权威配置源。
- 预览器不替代游戏内最终验收，也不模拟伤害、碰撞或 Buff。
- 不引入 Web 框架、第三方 CDN、上传服务或云端存储。
