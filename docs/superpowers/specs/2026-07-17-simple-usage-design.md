# 战斗特效 Skill 简化使用设计

## 目标

将普通使用流程收敛为一句自然语言调用。使用者不需要复制 JSON 模板、填写特效清单或手动运行校验命令。

## 用户入口

仓库主页只展示以下基础调用方式：

```text
使用 $creating-game-combat-vfx 为这组技能设计、制作并接入完整战斗特效。
```

同时增加一个完整示例，说明用户只需提供技能主题、表现要求和是否接入游戏；Skill 负责视觉设计、资源制作、接入、配置和验证。

## 内部流程

- Skill 根据任务规模判断是否需要结构化特效清单。
- 需要清单时，由 Skill 从内置模板自动创建并填写，不要求用户操作。
- Skill 自动运行清单校验，并自行修正可恢复的格式或字段问题。
- 模板与校验脚本继续保留，作为 Skill 内部的确定性工具。
- 只有调试 Skill 本身的开发者才需要直接使用模板或校验命令。

## 文档调整

- `SKILL.md`：将“复制模板并运行命令”改为自动生成和自动校验要求。
- `README.md`：基础使用区仅保留一句话入口和中文使用示例；将手动命令从普通流程移除。
- `README.en.md`：同步英文入口和示例。
- 仓库结构仍列出模板与校验脚本，并标明它们是内部资源。

## 验收标准

- 普通用户无需执行任何复制文件或命令行操作。
- 中英文主页均包含一句话调用和一个完整使用示例。
- 示例明确区分用户提供的信息与 Skill 自动完成的工作。
- `SKILL.md` 明确要求自动创建和校验清单。
- Skill 结构校验与现有清单校验脚本继续通过。
